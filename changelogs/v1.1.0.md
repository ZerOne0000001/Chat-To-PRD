# Chat-To-PRD v1.1.0 变动记录

## 核心工作流重构 (pm-workflow)
* 引入强约束的移交协议 (Handshake Protocol) 确保技能精准路由，杜绝通用模型的错误兜底。
* 修复了“快速通道”批量需求创建的目录顺序逻辑（先判定再生成目录）。
* 强制要求工作流在收尾“快速通道”时跳过交互 PRD 的询问，形成完整闭环。
* 增加迭代日志表头模板 (`Log_Format.md`) 规范输出格式，避免 AI 自创 Markdown 表格。

## 交互式 PRD 增强 (interactive-prd-builder)
* 彻底剥离旧版外部 assets（CSS/JS），强制要求生成基于 Tailwind CSS CDN 的单文件 HTML，降低后续用户的维护门槛。
* 新增 **PRD 内容区底部视觉补偿** 规则（`padding-bottom: 100vh`），彻底解决最后几个短章节无法滚动到屏幕顶部触发 iframe 联动死锁的问题。
* 将 Mermaid 极度脆弱的四重防御机制抽取到独立的 `references/mermaid-defense-patterns.md` 中，大幅降低主文件的上下文复杂度（Context Bloat）。

## 原型与设计优化 (pm-prototype & pm-design)
* **pm-design**：去除了对页面具体位置（顶部/底部等）和 UI 风格的过早定义，强制为下一阶段（原型阶段）保留足够的发散与创作空间。
* **pm-prototype**：
  * 强化逆向审计（Reverse Sync）流程，引入 Checklist 强制要求大模型审计 HTML 源码并更新 `Page_Structure.md`。
  * 引入 `Design_Trend_Guide.md` 强约束模板，强制大模型按极端风格预设参数，杜绝“AI塑料感”与“通用后台灰”。
  * 明确要求原型的 `Prototypes/` 文件夹必须且只能生成在当前需求根目录下。

## 合规与清理
* 严格执行 `skill-creator` 核心规范，清理了 `interactive-prd-builder` 中未被引用的幽灵资产。
* 删除了分布在各个子技能目录中的冗余 `LICENSE.txt`、`README.md` 等非必要说明文件，保持技能沙盒绝对干净。
