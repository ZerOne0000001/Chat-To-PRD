# Chat-To-PRD v1.2.0 变动记录

## PRD 生成强化 (pm-prd)
本次更新对 `pm-prd` 技能及其标准模板进行了深度的结构化重构，旨在彻底解决大模型生成 PRD 时容易出现的“逻辑遗漏”、“双端（前端/后端）逻辑混杂”以及“开发人员阅读负担重”的痛点。

### 1. 极致的排版与层级重构
* **引入页面缩进树**：废弃了 `mindmap` 思维导图，强制要求在第 6 章的页面总览中使用“纯文本缩进树（如 `├──` 符号）”，清晰直观地呈现“通用区域”与“状态特有区域”的包含关系。
* **隔离与视觉锚点**：强制将“页面概述”等前置信息包裹在 `> 引用块` 中；强制所有的 UI 元素名/按钮名必须加粗，所有的状态值/字段名必须包裹在代码块中，提升开发扫读（Skimming）效率。
* **弃用宽大表格**：在组件与元素拆解模块，**全面禁止使用表格**。改为使用带缩进的标签化列表，并强制打上 `[前端]` 和 `[数据]` 标签，实现展示逻辑与数据逻辑的彻底解耦。

### 2. 交互逻辑的三级解构 (防漏核心)
* 将原本平铺直叙的“交互与逻辑”重构为**以“交互元素”为核心锚点的三级嵌套结构**（交互元素 -> 触发动作 -> 前置校验/执行响应）。
* **明确视觉与交互双重阻断**：在描述前置校验时，强制拆分并交代“数据要求”、“视觉阻断（如按钮置灰）”和“交互阻断（如点击报错）”。
* **异常提示分类标准化**：规范了异常提示的交互层级，强制大模型必须在 PRD 中明确指定提示类型：
  * **弱提示 (Toast)**：轻量级错误（如“网络超时”），不阻断页面。
  * **强提示/阻断 (Modal/Alert)**：严重错误（如“库存不足”），需用户手动确认。
  * **二次确认 (Confirm)**：高危操作（如“取消订单”），防误触双按钮弹窗。

### 3. 全局防漏与 MECE 穷举
* **强制暗逻辑检查**：在分析假设阶段，强制大模型思考并补充页面上看不见的后台机制（如定时任务、过期时间、并发限制等），并在 PRD 示例的第 5 章中增加了【后台运行机制与触发器】模块。
* **状态穷举 (MECE)**：在描述任何核心组件时，强制要求覆盖 5 种基本状态：**正常态**、**空数据态**、**加载/过渡态**、**错误/断网态**、**极限/边界态**。
* **逆向联动影响 (Side Effects)**：强制要求在复杂交互动作的末尾，必须写明该操作引发的底层数据或跨页面的“联动影响”。

### 4. 跨章节锚点直达
* 为了提升阅读体验，强制要求在第 6 章引用第 3/4 章的实体、状态机或规则时，**必须提供 Markdown 内部锚点链接**（如 `[订单实体](#322-订单实体)`），并配合简要复述，避免开发频繁上下滚动查阅。
