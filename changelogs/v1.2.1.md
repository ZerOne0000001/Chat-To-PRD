# Chat-To-PRD v1.2.1 变动记录

## PRD 生成细节打磨 (pm-prd)
本次更新延续了 `v1.2.0` 的 PRD 结构化重构思路，进一步修正了页面级信息提示、树形结构渲染以及数据建模约束，重点解决“不同 Markdown 渲染器兼容性不一致”和“核心业务说明容易被忽略”的问题。

### 1. 页面级提示区块兼容性修正
* **放弃不稳定的扩展语法**：移除了对 GitHub Alerts 语法（如 `> [!IMPORTANT]`）的依赖，避免在不支持该扩展的 Markdown 渲染器中退化为普通文本。
* **放弃不稳定的内联红字方案**：尝试过的 `<font color="red">` 方案由于并非标准 Markdown 能力，现已废弃，避免在不同平台中出现兼容性偏差。
* **回归原生 Markdown 高兼容表达**：统一采用“引用块 + Emoji + 加粗标题”的方式来呈现页面级的“准入前置条件”和“全局异常兜底”，在兼容性、可读性和醒目程度之间取得平衡。

### 2. 页面状态树与排版规则校正
* **固定树形结构表现形式**：明确要求“页面区域与状态树”必须使用包裹在 `text` 代码块中的纯文本缩进树，确保 `├──`、`└──` 等层级结构在 Markdown 渲染时不会丢失。
* **修正错误示例方向**：纠正了围绕“是否使用代码块包裹树形结构”的反复试验结论，最终统一回到“必须包裹”的安全方案。
* **收敛页面级视觉表达**：对前置条件、异常兜底、页面概述和树形结构之间的相对位置进行了重新整理，减少过重的视觉打断，同时保证关键信息不被正文淹没。

### 3. 数据建模与 ER 关系约束增强
* **补强名词字典完整性要求**：在 `SKILL.md` 中新增硬性约束，明确“名词字典与实体属性”表格不可丢失任何业务字段。
* **锁定模板列结构**：要求字段说明表格必须严格遵循模板中的列结构：`字段名称`、`字段类型`、`限制/长度`、`必填`、`业务含义`，避免大模型自由发挥。
* **强化中文产品视角**：进一步强调实体名、字段名、属性名必须使用中文命名，禁止混入开发变量名。
* **ER 图关系显式化**：要求在实体关系图中必须明确体现业务关系及基数，例如 `1:N`，并配合关系说明（如“下单 (1:N)”）。

### 4. 示例文档与规范文件同步收口
* **同步修正 `PRD_Example.md`**：将页面级概述区块、状态树、前置条件和异常兜底的写法统一到最新规范。
* **同步修正 `PRD_Component_Standard.md`**：继续对组件拆解标准进行细节对齐，确保技能规则、参考标准和示例文档之间不再互相打架。
* **强化自检口径**：`SKILL.md` 中的自我审查清单已同步更新，确保模型在交付前能用统一标准检查排版、逻辑与数据建模完整性。

## 交互式 PRD 体验增强 (interactive-prd-builder)

### 1. 原型独立预览入口补强
* **新增“新标签页打开”能力**：在 `interactive-prd-builder` 中增加了右侧 iframe 原型预览区右上角的图标按钮要求，允许用户直接在新标签页中打开当前正在预览的 HTML 原型页面。
* **同步更新按钮链接逻辑**：在 `references/sync-patterns.md` 中补充了按钮 `href` 与 iframe `src` 同步更新的代码模式，避免按钮打开的页面与当前预览状态不一致。

## 总结
`v1.2.1` 不是一次大而全的新功能发布，而是一次非常关键的“规范收口 + 兼容性修正”更新。它让 `pm-prd` 的排版策略、数据建模约束和示例表现形式更加统一、可执行，也让 `interactive-prd-builder` 的原型预览体验更顺手。
