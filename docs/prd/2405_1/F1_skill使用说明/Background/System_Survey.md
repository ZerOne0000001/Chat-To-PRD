# 系统上下文与调研分析 (System Survey)

## 1. 核心目标与受众 (提取自背景文档)
*   **核心目标**：提供一个结构化、类似 PPT 形式的“技能使用说明/大纲”独立网页，用于在视频录制时作为讲解串场指引，降低小白用户的认知门槛。
*   **受众群体**：讲解人（需要在录屏中频繁切换展示进度）以及视频观众（需要直观看到当前处于什么阶段、产出了什么文件）。
*   **关键场景**：录屏前的全局架构概览；录屏中频繁的“IDE 对话框”与“大纲页面”之间的来回切屏展示。

## 2. 架构定调：全新的独立系统/页面
*   **用户确认**：这是一个**全新的独立页面**。
*   **原有系统逻辑**：无历史包袱。不需要依附于任何现有的 IDE（如 Trae/Cursor）的插件体系，也不需要直接在 IDE 侧边栏进行集成。
*   **受影响的页面**：无。

## 3. 约束与依赖 (Constraints & Dependencies)
*   **展示约束**：因为要配合视频录制（可能全屏或半屏录制），页面的视口（Viewport）布局必须极度清晰，能够承受频繁的窗口切换而不丢失当前讲解焦点的上下文。
*   **未来扩展预判**：作为一个“说明”页面，未来可能会从单一的 `pm-workflow` 扩展到讲解其他技能（如 `pm-design` 或 `frontend-design`）。在底层结构设计上，应当预留“切换不同技能说明”的扩展性，但本次 MVP 版本先聚焦于 `pm-workflow`。
