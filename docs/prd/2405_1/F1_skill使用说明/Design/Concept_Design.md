# 概念设计与方案决策 (Concept Design)

## 1. 方案推演：演示大纲的交互骨架

针对“作为视频演示串场 PPT”的核心诉求，我们在前期调研中提出了两种完全不同的交互概念：

### 方案 A：沉浸式“步骤滚动/锚点高亮”单页流 (Single-page ScrollSpy)
*   **交互机制**：左侧悬浮全局大纲目录，右侧为全长内容瀑布流。用户通过滚动或点击左侧锚点定位到对应的阶段。
*   **优点 (Pros)**：
    *   全局视野最佳，用户随时知道上下文进度。
    *   符合传统文档的阅读习惯。
*   **缺点 (Cons)**：
    *   页面较长时，右侧信息量过载，容易让观众失焦。
    *   视频录制切屏时，滚动的过程可能显得凌乱。

### 方案 B：PPT式“幻灯片切页/步骤条”卡片流 (Slide/Stepper Focus)
*   **交互机制**：页面主体区域每次**仅聚焦展示一个阶段的内容**（例如：只显示“阶段 1：需求启动”及其对应的产物）。全局的流程进度通过顶部或侧边的“步骤条 (Stepper)”来指示。
*   **优点 (Pros)**：
    *   视觉焦点极其纯粹，完美契合视频讲解时“步步为营”的节奏。
    *   信息降噪，小白观众认知成本最低。
*   **缺点 (Cons)**：
    *   相比长页面，缺乏直接的全局前后文连贯感。
    *   如果步骤过多，用户可能无法快速找到之前的某个环节。

## 2. 最终方案决策 (Final Decision)

**结论：选择 方案 B (带快速跳转的幻灯片卡片流)。**

**决策理由与优化说明**：
为了保证视频演示中“切回大纲”那一瞬间的视觉冲击力和焦点纯净度，方案 B 是最佳选择。为了弥补其“无法快速找到之前环节”的缺点，我们对方案 B 进行了核心交互优化：

*   **优化：全局进度导航器 (Global Step Navigator)**：
    在页面的顶部（或左侧固定区域），必须设计一个全局可见的“步骤条”或“目录索引”。这个导航器不仅用于指示当前进度（例如高亮当前进行到的 `阶段 3`），还必须具备**快速点击跳转 (Quick Jump)** 的能力。
*   **实际录屏场景推演**：演示者在讲到“阶段 4”时，突然想回顾“阶段 1”生成的背景文档。他只需在大纲页面的全局导航器上点击“阶段 1”的图标，页面主体内容瞬间平滑切换回阶段 1 的卡片，无需前后翻页寻找。

## 3. 概念方案的视觉意向
*   **布局骨架**：顶部是贯穿的进度导航（步骤 0 ~ 步骤 6），下方居中是巨大的内容展示卡片（Card）。
*   **卡片内部**：卡片标题为当前阶段名称，卡片正文分为左右两块：左边是该阶段的触发动作或简介，右边是该阶段生成的中间产物列表（文件名+一句话介绍）。
