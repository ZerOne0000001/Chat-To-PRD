# 信息架构与页面结构 (Page Structure)

## 全局导航与结构概览
由于该需求定位为一个独立的单页面 Web 演示大纲，不涉及复杂的多页面跳转，所有内容将在同一个 HTML 文件中呈现。
- **目标端口**：桌面端 Web (PC 端为主，适配录屏比例)
- **主页面文件**：`index.html` (或 `Interactive_PRD.html`)

## 页面分区拆解 (`index.html`)

### 1. 全局进度导航器 (Global Step Navigator)
*   **功能定位**：指示当前演示进度，提供全局视角的快速跳转能力。
*   **视觉抽象**：横跨页面顶部的步骤条 (Stepper)，或者悬浮在左侧的索引菜单。
*   **包含元素**：
    *   **Logo/Title**：`Chat-To-PRD (pm-workflow)`。
    *   **步骤节点 (Steps)**：根据选择的场景动态渲染。
        *   标准工作流：`[0] 初始化` -> `[1] 背景` -> `[2] 架构设计` -> `[3] 原型制作` -> `[4] 文档生成` -> `[5] 迭代收尾` -> `[6] 交互式 PRD`。
        *   快速通道：`[0] 初始化` -> `[4] 文档生成` -> `[5] 迭代收尾`。
*   **交互动作**：
    *   **高亮指示**：当前正在展示的阶段节点必须处于 Active 激活态（如加粗、高亮色）。
    *   **点击跳转**：点击任意节点，页面主体瞬间切换至该阶段的内容卡片。

### 2. 主体展示卡片区 (Main Content Slide)
*   **功能定位**：PPT 式的幻灯片区域，每次仅聚焦展示一个阶段的核心信息。
*   **包含元素与布局**：
    *   **卡片头部**：当前阶段名称（例如：`背景 & 调研`）及辅助序号说明（例如：`第 1 阶段`）。
    *   **卡片左侧 (动作简介)**：简要描述 AI 在该阶段执行的动作（如“系统执行深度访谈协议，在架构设计之前，明确业务目标、用户画像与核心场景”）。
    *   **卡片右侧 (产出物列表)**：列出该阶段生成的中间文件。
        *   **文件项 (File Item)**：包含特定图标（基于产物类型动态渲染，如文档、列表、地图、结构图等）、文件名称（如 `Requirement_Background.md`）、以及一句话的文档描述。
    *   **底部控制栏 (新增)**：为方便演示，提供“← 上一步”与“下一步 →”的快捷切换按钮。
*   **交互动作**：
    *   **卡片切换**：配合顶部导航器或底部控制栏的点击，该区域内容需执行平滑的切页动画（如淡入向上浮现 `fade-enter` 与背景光晕渐变）。

### 3. 场景选择与初始化区 (Scenario Selector)
*(该区域为演示过程的“阶段 0”的展示内容)*
*   **功能定位**：展示技能基础信息，并提供分支路线的选择。
*   **包含元素**：
    *   **技能简介**：大字号的技能名称（`pm-workflow`）与说明（“全流程产品设计向导，负责迭代生命周期管理并按需调度子技能。请选择您当前的需求流转场景：”）。
    *   **场景切换开关 (Cards)**：两个巨大的选项卡或按钮：`标准工作流 (Standard)` 与 `快速通道 (Fast Track)`，选中时有明显的发光/高亮反馈。
    *   **开始演示按钮 (新增)**：位于场景卡片下方，点击后正式进入该场景的第一步。
*   **交互动作**：
    *   **点击切换场景**：点击任意一个分支，标记为激活状态，并更新内部状态，但不会立即跳转。
    *   **点击开始演示 (新增)**：点击后触发顶部的“全局进度导航器”重载，并执行下一步跳转动画。

### 4. 前置准备区 (Preparation Area) [新增]
*(该区域为进入页面的默认首屏，即“阶段 00 准备工作”的展示内容)*
*   **功能定位**：在进入核心业务流前，指导用户配置必备的环境、插件与底层工具。
*   **视觉抽象**：支持上下滚动的长篇文档式卡片（带有自定义样式的滚动条）。
*   **包含元素**：
    *   **模块一：编辑器工具的选择与要求**：说明为什么需要 AI IDE，阐述三个核心要求（操作资源管理器、并发多任务、自定义工作流），并推荐 Trae IDE、通义灵码、腾讯云AI代码助手、MarsCode、DeepSeek IDE 等国产工具，以及 Cursor、Windsurf、Project IDX(Gemini)、Claude Code、GitHub Copilot 等国外优秀工具。
    *   **模块二：必须提前安装的环境与插件**：
        *   **Node.js (npm)**：说明用于执行一键安装指令的必要环境。
        *   **Live Server**：强调安装 Live Server 的必要性（解决同源策略 CORS 限制）及使用方法。
    *   **模块三：核心 Skill 的安装与配置**：展示项目级 (Project) 与全局级 (Global) 不同的安装作用域及对应的存放目录（明确区分 macOS/Linux 与 Windows 系统的路径，并强调“隐藏文件夹名称通常就是软件名称”），展示完整的 Skill 项目树形结构示例。提供 `npx skills add` 的一键安装指令，以及如何通过 AI Chat 对话框验证安装是否成功的具体提示词。
*   **交互动作**：
    *   **滚动条 (Scrollbar)**：鼠标悬停在内容区时，可平滑滚动阅读长文，自定义滚动条轨道与滑块颜色。
    *   **底部固定控制栏**：提供 `完成准备，下一步 →` 按钮，点击进入场景选择（阶段 0）。
