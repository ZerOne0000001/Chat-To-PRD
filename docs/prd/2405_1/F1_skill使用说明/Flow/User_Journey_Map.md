# 用户旅程地图 (User Journey Map)

## 演示者视角的“视频录制与大纲切换”旅程

| 阶段 | 1. 视频开场导入 | 2. 工作流初始化与触发 | 3. 阶段串讲与切换 (核心循环) | 4. 产物展示与总结 |
| :--- | :--- | :--- | :--- | :--- |
| **用户行为** | 演示者打开“skill使用说明大纲页面”，向观众介绍今天要演示的 `pm-workflow` 技能。 | 演示者切回 AI IDE（如 Trae），输入一句话需求触发主控技能，并开始走标准工作流。 | AI 在 IDE 中跑完某个子技能（如 `pm-kickoff`），演示者切回大纲页面，向观众讲解刚才发生了什么，然后再切回 IDE 确认并进入下一阶段。 | 演示者在大纲页面展示最后生成的 PRD，并对整个工作流进行串联复盘。 |
| **用户痛点** | 之前直接干讲没有结构，观众记不住技能全貌。 | 观众可能不知道 AI 是如何被唤起的，以及不同输入会导致的分支。 | 频繁切换会导致观众迷失焦点，不知道当前进度在哪。 | 如果没有回顾大纲，观众容易忘记前面生成的众多中间文档是什么。 |
| **系统支持 (Touchpoints)** | **大纲页面：技能结构、简介与触发条件模块。** | **大纲页面：使用场景分支说明模块。** | **大纲页面：阶段大纲高亮（类似进度条/当前步骤指示）。** | **大纲页面：每个阶段对应的产物名称及文档介绍列表。** |
| **情绪曲线** | 😐 平和 | 🙂 期待 | 😅 容易混乱，依赖大纲找回焦点 | 😄 豁然开朗，理解体系 |
| **机会点** | 设计一个极其清晰的页面顶部区域，突出技能核心价值。 | 明确用两列或对比方式展示“标准工作流”和“快速通道”的差异。 | 在页面侧边或顶部提供一个固定且明显的步骤导航器，演示者讲到哪，哪里就高亮。 | 在大纲中采用简洁的列表或卡片形式罗列产物，拒绝长篇大论，仅需名称和一句话介绍。 |
