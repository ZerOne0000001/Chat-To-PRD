# 页面结构：移动端自助收银系统 (Mobile Cashier)

## 全局导航与结构
本应用以“食堂自助极速核销”为核心生命周期，主要包含以下核心页面/模块：
1. **首页/待机页 (Home)** `home.html`
2. **识别过度页 (Identify)** `identify.html`
3. **支付结果页 (Result)** `result.html`
4. **后台管理台 (Admin Dashboard)** `admin_dashboard.html`

---

## 1. 首页/待机页 (Home) `home.html`
- **顶部状态栏 (Header)**
  - 左侧：时间展示 (如 12:00) 和网络状态图标 (Wi-Fi/4G)。
  - 右侧：当前档口名称。
- **核心信息区 (Hero Section)**
  - **餐次标题**：显示当前餐次，如“午餐” (特大字号)。
  - **时段提示**："11:00 - 13:00" (辅助说明)。
  - **价格轮播**："标准价 ￥15.00" / "教工价 ￥10.00"。
- **主交互区 (Action Area)**
  - **默认状态**：中心区域展示动态引导插画 "请刷卡、扫码或刷脸"。
  - **快捷按钮**：底部悬浮 [刷脸支付] 圆形大按钮。
- **全局组件**：隐藏式的密码呼出热区（连点右上角5次弹出后台登录框）。

## 2. 识别过度页 (Identify) `identify.html`
- **中心动画区**：
  - 显示雷达扫描动画或摄像头实时画面。
  - 文本提示："正在识别身份..."。
- **底部操作栏**：
  - [取消并返回首页] 按钮（倒计时 10 秒后若未识别成功自动触发）。

## 3. 支付结果页 (Result) `result.html`
- **状态提示区 (Toast/Hero)**
  - 成功：✅ 绿色大图标，特大号金额 "￥10.00"，同步触发“支付成功”语音播报。
  - 失败：❌ 红色大图标，提示失败原因（如余额不足）。
- **明细卡片 (Details Card)**
  - 姓名: "张三" (脱敏显示)
  - 类型: "教工"
  - 余额: "￥120.50" (余额低于20元时，数字标红并闪烁)。
- **交互流 (Interaction Flow)**
  - 页面不提供主动返回按钮。
  - 驻留 3 秒后，自动返回 `home.html`。
- **弹窗组件 (Modal: 二次确认防呆)**
  - **触发条件**：同一餐次内重复刷卡。
  - **表现形式**：屏幕居中弹出遮罩层。
  - **内容**："您在[午餐]已消费[1]次，确认再次支付？"
  - **按钮组**：[确认支付] (蓝色高亮) / [取消] (灰色，5秒无操作自动触发)。

## 4. 后台管理台 (Admin Dashboard) `admin_dashboard.html`
- **侧边导航栏 (Sidebar)**：包含“交易流水”、“餐次规则”、“异常处理”三个 Tab。
- **主内容区 (异常处理 Tab)**：
  - 顶部：搜索栏（按学号/时间筛选）。
  - 数据表格：列出异常订单（如重复扣费申诉）。
  - **操作列**：包含 [一键退款] 和 [标记已处理] 按钮。点击退款后弹出二次确认框，确认后行状态变为“已退款”。
