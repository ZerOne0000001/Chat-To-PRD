# PRD 组件与模块拆解标准参考 (Component & Module Breakdown Standard)

本指南用于规范在 PRD 第 6 章“功能模块与页面细节”中，如何拆解和描述复杂的交互组件（如键盘、列表、抽屉、二维码等）。
**原则：绝不能用一句话敷衍带过，必须从逻辑、来源、格式、异常等全方位拆解。**

---

## ❌ 错误示范 (Anti-Pattern)

**场景**：描述一个带有录入数字键盘的弹窗。
**错误写法**：
> 录入比分弹窗底部有一个数字键盘，用户可以点击键盘输入分数。如果输错可以点删除。

**为什么错？**
- 缺少结构化表格，无法让开发一眼看出元素。
- 缺少具体按键定义（0-9怎么处理？小数点？正负号？）。
- 缺少业务逻辑与异常兜底（输入 0 会怎样？删光了会怎样？）。

---

## ✅ 正确示范 (Best Practice)

**场景**：描述带有数字键盘的录入比分抽屉。

#### 6.2.X 【弹窗】录入比分全屏抽屉与数字键盘

- **区域介绍与规则**：高度为 95% 的全屏抽屉。纵向列表展示所有人，底部固定一个类似计算器的自定义数字键盘，用于快速录入单局分数。
- **展示元素定义**：

| 元素名称 | 逻辑 (数据来源/计算逻辑) | 限制与格式 |
| :--- | :--- | :--- |
| 玩家输入框 | `用户录入数值` | 点击任意行，该行的输入框变为“激活态”（带特定边框和底色），其他行变为半透明。 |
| 校验差值 | `所有输入框数值的总和` | 若总和 ≠ 0，显示具体差值（如 +15 或 -20）。若总和 = 0，显示绿色“已平账”。 |

- **自定义数字键盘按键逻辑说明**：
  - **数字键 (0-9)**：点击后将数字追加到当前激活的输入框。若当前值为 0，则直接覆盖。
  - **退格键 (⌫)**：删除当前输入框的最后一位字符。若全部删除，自动恢复为 `0`。
  - **正负号键 (+/-)**：用于切换当前输入数值的正负状态。
  - **下一位键 (⬇️)**：点击后激活态下移到下一个玩家。如果当前为最后一个玩家，则循环回到第一个玩家。
  - **提交大按钮**：跨两行高度的大按钮。平账时为绿色高亮激活，不平账时半透明置灰。

- **区域交互 (自动赋分逻辑)**：
  - **前置条件**：抽屉打开，默认激活第一个玩家。
  - **触发动作**：点击键盘“下一位”键跳过某人或完成输入。
  - **结果呈现**：
    1. **跳过某人**：值若为空，默认为 0。
    2. **自动补齐**：如果切换到**最后一个人**，系统自动计算前 N-1 个人的分数总和并取反填入最后一人，实现自动平账。

- **异常与兜底**：
  - 当校验差值不为 0 时，“提交”按钮强制置灰阻断，点击无响应。
